# Implementation Plan: Plugin API Extension for Creating New Resource Types

**Date:** 2026-05-27  
**Goal:** Enable Crane plugins to generate new Kubernetes resources (not just transform existing ones)  
**Use Case:** BuildConfig → Shipwright Build conversion  
**Approach:** V1 API Extension (no V2 needed) - 100% backward compatible

---

## Context and Motivation

### Current State
The Crane plugin system allows plugins to only:
- **Transform** existing resources using JSONPatch operations
- **Mark resources for deletion** using the whiteout flag

**Limitation:** Cannot create new resource types that didn't exist in the input data.

### Requirement
Enable plugins to generate new resources for scenarios like:
- **BuildConfig → Shipwright Build** - OpenShift to Kubernetes-native conversion
- **DeploymentConfig → Deployment** - deprecated resource migration
- **ImageStream → container registry references** - OpenShift dependency resolution

### Architecture
```
Input Resources
    ↓
[Orchestrator] → [Runner] → [Plugin]
    ↓                           ↓
RunnerResponse.NewResources ←─┘
    ↓
TransformArtifacts (original + new)
    ↓
[Writer] → Kustomize structure
```

**Key Insight:** The existing Writer already handles artifacts uniformly, no changes needed!

## Implementation Phases

### **PHASE 1: crane-lib API Extension** (tasks #1, #2, #4)

#### 1.1 Extend PluginResponse
**File:** `../crane-lib/transform/plugin.go` (lines 29-33)

```go
type PluginResponse struct {
    Version      string                      `json:"version,omitempty"`
    IsWhiteOut   bool                        `json:"isWhiteOut,omitempty"`
    Patches      jsonpatch.Patch             `json:"patches,omitempty"`
    
    // NEW: Additional resources to be generated by the plugin
    NewResources []unstructured.Unstructured `json:"newResources,omitempty"`
}
```

**Rationale:**
- `omitempty` tag ensures backward compatibility
- `[]unstructured.Unstructured` is the standard K8s resource representation
- Each new resource becomes a separate artifact

**Backward Compatibility Note:**
- The `omitempty` tag ensures old plugins (without NewResources) work unchanged
- Go JSON unmarshaling automatically sets an empty slice `[]` when field is missing
- Old crane versions ignore new JSON fields (graceful degradation)
- **NO NEED to create V2 version** - V1 extension is fully compatible!

#### 1.2 Extend RunnerResponse
**File:** `../crane-lib/transform/runner.go` (lines 24-31)

```go
type RunnerResponse struct {
    TransformFile  []byte
    HaveWhiteOut   bool
    IgnoredPatches []byte
    
    // NEW: Resources generated by plugins
    NewResources   []unstructured.Unstructured
}
```

#### 1.3 Update Runner.Run() Logic
**File:** `../crane-lib/transform/runner.go` (lines 62-124)

**Key Changes:**
1. Add `newResources := []unstructured.Unstructured{}` collection
2. In plugin loop, collect `resp.NewResources`
3. Return NewResources in RunnerResponse
4. **NO version validation needed** - detection via `len(NewResources) > 0`

**Simplified Implementation (without V2 version checks):**
```go
func (r *Runner) Run(object unstructured.Unstructured, plugins []Plugin) (RunnerResponse, error) {
    haveWhiteOut := false
    havePatches := false
    patches := []PluginOperation{}
    newResources := []unstructured.Unstructured{}  // NEW
    errs := []error{}

    for _, plugin := range plugins {
        c := object.DeepCopy()
        resp, err := plugin.Run(PluginRequest{Unstructured:*c, Extras:r.OptionalFlags})
        if err != nil {
            errs = append(errs, err)
            continue
        }
        
        if resp.IsWhiteOut {
            haveWhiteOut = true
        }
        
        if len(resp.Patches) > 0 {
            havePatches = true
            patches = append(patches, PluginOperationsFromPatch(plugin.Metadata().Name, resp.Patches)...)
        }
        
        // NEW: Collect new resources - NO version check!
        // If plugin sends NewResources, we simply collect them
        if len(resp.NewResources) > 0 {
            newResources = append(newResources, resp.NewResources...)
            r.Log.Debugf("Plugin %s generated %d new resource(s)", 
                plugin.Metadata().Name, len(resp.NewResources))
        }
    }
    
    response := RunnerResponse{
        TransformFile:  []byte(`[]`),
        HaveWhiteOut:   haveWhiteOut,
        IgnoredPatches: []byte(`[]`),
        NewResources:   newResources,  // NEW
    }
    
    // ... existing error handling and patch sanitization ...
    
    return response, nil
}
```

**Why it works without version checks:**
- Old plugins never send NewResources → `len() == 0` → ignored
- New plugins send NewResources → `len() > 0` → processed
- Go unmarshaling automatically handles compatibility via `omitempty`

### **PHASE 2: crane Orchestrator Integration** (tasks #3, #5)

#### 2.1 Update Orchestrator.transformResources()
**File:** `internal/transform/orchestrator.go` (lines 199-247)

**Key Changes:**
1. After creating artifact for original resource, iterate through `response.NewResources`
2. For each new resource, create a separate `TransformArtifact`
3. Set `HaveWhiteOut: false` (new resources are always active)
4. Set `Patches: nil` (no patches on creation)
5. Validate that new resources have kind and name
6. Log new resource creation

**Implementation:**
```go
func (o *Orchestrator) transformResources(
    stage Stage, 
    stagePlugin cranelib.Plugin, 
    inputResources []unstructured.Unstructured,
) ([]cranelib.TransformArtifact, error) {
    // Build plugins list
    var plugins []cranelib.Plugin
    if stagePlugin != nil {
        plugins = []cranelib.Plugin{stagePlugin}
    }

    runner := cranelib.Runner{
        Log:              o.Log,
        PluginPriorities: nil,
        OptionalFlags:    o.OptionalFlags,
    }

    var artifacts []cranelib.TransformArtifact

    for _, resource := range inputResources {
        response, err := runner.Run(resource, plugins)
        if err != nil {
            resourceID := o.formatResourceID(resource)
            return nil, fmt.Errorf("stage %s: failed to run transform for %s: %w", 
                stage.DirName, resourceID, err)
        }

        // Parse patches for original resource
        var patches jsonpatch.Patch
        if len(response.TransformFile) > 2 && !response.HaveWhiteOut {
            patches, err = jsonpatch.DecodePatch(response.TransformFile)
            if err != nil {
                resourceID := o.formatResourceID(resource)
                return nil, fmt.Errorf("stage %s: failed to decode patches for %s: %w", 
                    stage.DirName, resourceID, err)
            }
        }

        // Create artifact for the original resource
        artifact := cranelib.TransformArtifact{
            Resource:     resource,
            HaveWhiteOut: response.HaveWhiteOut,
            Patches:      patches,
            IgnoredOps:   []cranelib.IgnoredOperation{},
            Target:       cranelib.DeriveTargetFromResource(resource),
            PluginName:   stage.PluginName,
        }
        artifacts = append(artifacts, artifact)

        // NEW: Create artifacts for each new resource
        for i, newResource := range response.NewResources {
            // Validate that new resource has required metadata
            if newResource.GetKind() == "" || newResource.GetName() == "" {
                o.Log.Warnf("Stage %s: skipping new resource #%d with missing kind or name", 
                    stage.DirName, i)
                continue
            }
            
            newResourceID := o.formatResourceID(newResource)
            o.Log.Infof("Stage %s: plugin generated new resource: %s", 
                stage.DirName, newResourceID)
            
            newArtifact := cranelib.TransformArtifact{
                Resource:     newResource,
                HaveWhiteOut: false,  // Always active
                Patches:      nil,    // No patches initially
                IgnoredOps:   []cranelib.IgnoredOperation{},
                Target:       cranelib.DeriveTargetFromResource(newResource),
                PluginName:   stage.PluginName,
            }
            artifacts = append(artifacts, newArtifact)
        }
    }

    return artifacts, nil
}
```

```go
package main

import (
    "encoding/json"
    
    "github.com/konveyor/crane-lib/transform"
    "github.com/konveyor/crane-lib/transform/cli"
    buildv1 "github.com/openshift/api/build/v1"
    shipwrightv1beta1 "github.com/shipwright-io/build/pkg/apis/build/v1beta1"
    "k8s.io/apimachinery/pkg/apis/meta/v1/unstructured"
)

func main() {
    fields := []transform.OptionalFields{
        {
            FlagName: "search-registries",
            Help:     "Comma-separated list of search registries",
            Example:  "docker.io,quay.io",
        },
        {
            FlagName: "insecure-registries",
            Help:     "Comma-separated list of insecure registries",
            Example:  "localhost:5000",
        },
    }
    cli.RunAndExit(cli.NewCustomPlugin("BuildConfigPlugin", "v1", fields, Run))
}

func Run(request transform.PluginRequest) (transform.PluginResponse, error) {
    response := transform.PluginResponse{
        Version:      "v1",  // Still V1 - extension is backward compatible!
        IsWhiteOut:   false,
        Patches:      nil,
        NewResources: []unstructured.Unstructured{},
    }
    
    // Check if this is a BuildConfig
    if request.GetKind() != "BuildConfig" || 
       request.GetAPIVersion() != "build.openshift.io/v1" {
        // Not a BuildConfig - pass through unchanged
        return response, nil
    }
    
    // This IS a BuildConfig - convert it
    
    // 1. Mark original BuildConfig for deletion
    response.IsWhiteOut = true
    
    // 2. Parse BuildConfig
    var buildConfig buildv1.BuildConfig
    requestBytes, _ := request.MarshalJSON()
    json.Unmarshal(requestBytes, &buildConfig)
    
    // 3. Convert to Shipwright Build
    shipwrightBuild, err := ConvertBuildConfigToShipwright(&buildConfig, request.Extras)
    if err != nil {
        return response, err
    }
    
    // 4. Convert to unstructured
    buildBytes, _ := json.Marshal(shipwrightBuild)
    var unstructuredBuild unstructured.Unstructured
    json.Unmarshal(buildBytes, &unstructuredBuild)
    
    // 5. Add to new resources
    response.NewResources = append(response.NewResources, unstructuredBuild)
    
    return response, nil
}
```

## Edge Cases and Solutions

### 1. Duplicate Resource Names
**Problem:** Plugin generates resource with same kind/namespace/name as existing.

**Solution:** Writer already handles duplicates via deduplication (writer.go lines 73-98):
- If existing is whiteout and new is active: replace with active
- If both same type: last one wins (with warning)

**Recommendation:** Plugin authors should ensure unique names.

### 2. Cross-Resource References
**Problem:** New resource references original resource being deleted.

**Example:** BuildConfig references ImageStreamTag, Shipwright Build needs resolved image URL.

**Solution:** Plugin must resolve references during conversion:
```go
// Plugin resolves ImageStreamTag reference
imageRef := resolveImageStreamTag(bc.Spec.Output.To.Name, request.Extras)

build := &shipwrightv1beta1.Build{
    Spec: shipwrightv1beta1.BuildSpec{
        Output: shipwrightv1beta1.Image{
            Image: imageRef,  // Fully resolved!
        },
    },
}
```

### 3. Patching Generated Resources
**Problem:** User wants to patch a generated resource in a later stage.

**Solution:** Works naturally - generated resources flow through pipeline:
```bash
# Stage 30: Generate Shipwright Build
crane transform --stage 30_BuildConfigPlugin

# Stage 40: Patch the generated Build
crane transform --stage 40_CustomTweaks
```

Stage 40 sees the Build as a regular input resource and can patch it.

### 4. Large Conversions (100+ resources)
**Problem:** Processing 100 BuildConfigs, each generating a Shipwright Build.

**Current Behavior:**
- Runner called 100 times (once per resource)
- 200 artifacts collected (100 original + 100 new)
- Written to disk in batch

**Performance:** Linear O(n) - acceptable for most use cases.

## Backward Compatibility

### ✅ IMPORTANT: V2 Version Not Needed!

Thanks to Go JSON marshaling with `omitempty` tag, the extension is **100% backward compatible**:

**Old Plugin** (doesn't send NewResources):
```json
{
  "version": "v1",
  "isWhiteOut": false,
  "patches": []
}
```
- Go unmarshaling sets `NewResources` to empty slice `[]`
- Runner detects `len(NewResources) == 0` and ignores
- **Works unchanged!**

**New Plugin** (sends NewResources):
```json
{
  "version": "v1",
  "isWhiteOut": true,
  "newResources": [...]
}
```
- Still uses V1 protocol
- Runner detects `len(NewResources) > 0` and processes
- **No need to declare V2!**

### Compatibility Proof

**Test proved 3 scenarios:**
1. ✅ Old plugin → New crane: `NewResources` is `[]`, everything works
2. ✅ New plugin → New crane: `NewResources` contains data, everything works
3. ✅ New plugin → Old crane: Extra fields ignored (graceful degradation)

### Automatic JSON Compatibility

**Example - Old plugin response** (without NewResources):
```json
{
  "version": "v1",
  "isWhiteOut": false,
  "patches": []
}
```
→ Go unmarshal sets `NewResources: []` automatically

**Example - New plugin response** (with NewResources):
```json
{
  "version": "v1",
  "isWhiteOut": true,
  "newResources": [
    {
      "apiVersion": "shipwright.io/v1beta1",
      "kind": "Build",
      "metadata": {"name": "myapp-build"},
      "spec": {}
    }
  ]
}
```
→ Go unmarshal populates `NewResources` with data

**Key Points:** 
- `omitempty` tag = field doesn't need to be in JSON
- Go zero value = empty slice when field is missing
- Detection via `len() > 0` instead of version flags

## Summary

This implementation plan provides a **minimal, 100% backward compatible** extension to the Crane plugin API.

### 🎯 Key Insights:

1. **V2 version not needed!** 
   - Go JSON `omitempty` ensures automatic compatibility
   - Detection via `len(NewResources) > 0` instead of version flags
   - Simpler implementation = faster development

2. **Writer needs no changes**
   - Already handles artifacts uniformly
   - Doesn't distinguish artifact source (transformation vs new)

3. **Minimal code changes:**
   - Add `NewResources` field to 2 structures
   - Add collection in Runner (5 lines of code)
   - Add artifact creation in Orchestrator (15 lines)

### ✅ What We're Implementing:

1. **Extend API:** Add `NewResources []unstructured.Unstructured` with `omitempty`
2. **Collect Resources:** Update Runner to aggregate new resources
3. **Create Artifacts:** Update Orchestrator to create artifacts for new resources
4. **Test Thoroughly:** Ensure backward compatibility and sequential consistency

### 🚀 Advantages of This Approach:

- ✅ **100% backward compatible** - old plugins work unchanged
- ✅ **Simpler implementation** - no version negotiation complexity
- ✅ **Faster development** - 3-4 weeks instead of 4-5 weeks
- ✅ **Graceful degradation** - old crane ignores NewResources
- ✅ **Type-safe** - uses standard `unstructured.Unstructured`
- ✅ **Testable** - JSON marshaling/unmarshaling automatically tests compatibility

This approach follows existing patterns in crane-lib and enables powerful resource conversion scenarios like BuildConfig → Shipwright Build.
